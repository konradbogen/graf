graf

This page demonstrates the different options one can choose from when using the graf bundle.

graf.graph


Lets start off with the basic graph structure - some heads, some lines.

graf.filesystem


It is very unhandy to enter the graph structure inline - let's activate the FileSystem!

graf.visual.clicktoenter


Ooh, the structure got bigger. Let's add optionality to open children heads by clicking on a head.

graf.ui.zoom


Looks like we need an option to drag the map, now that it is even bigger.

graf.frame


How about giving members the option to open their own content in a seperate framing page?

graf.visual.mousehover-graf.visual.audio


Since the graph is very much about sound, let's add a "Mouse-Hover" input to play audio!

graf.pac


Let's try recording and playing sequences by adding a _record and a _play head. By the way, PAC stands for "Player Attention Control".

graf.graph.perm-graf.pac


To permutate all seqeuences, let's add a _perm head!
Saying that, the graf structure is a standardized "Graph" Data Structure, so it is possible to write own data restructuring algorhythms as well as machine learning implementation.

graf.ui.menu


Let the members decide, how their graf should look like by enabling the UI Menu.
Typing "xxx.xxx" creates a head and a children head, typing "xxx-yyy" creates a line between two heads.

graf.filesystem-graf.ui.header


The FileSystem in combination with the UI Header enables the members to store their graf by typing a name in the Header and pressing Enter.

graf.ui.menu-graf.ui.header


Let's enable input of an own graph and storing it by entering a name in the Header!

graf.visual.lightmode


Finally, let's brighten up things a little by typing >brightlight instead of >darkuniverse in the UI Input!
